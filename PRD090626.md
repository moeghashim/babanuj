## Crawl Report Triage

Initial parsed findings:

- `Error-Image_file_size_too_large.csv`: 44 oversized optimized image URLs, mostly Shopify product images requested through `/_next/image` at large widths such as `w=3840`.
- `Error-Image_file_size_too_large-links.csv`: 85 source-to-image links showing oversized assets surfaced from product, collection, brand, and home pages.
- `Error-indexable-Orphan_page_(has_no_incoming_internal_links).csv`: one indexable product page with zero internal href inlinks: `/product/bal-coffee-s-eastern-brew-12oz-of-tradition-in-every-cup-beverage-flavor`.
- `Notice-indexable-Page_has_only_one_dofollow_incoming_internal_link.csv`: five products with only one dofollow internal inlink.
- `Notice-indexable-Page_has_nofollow_outgoing_internal_links.csv` and related link export: 78 pages link to `/account/sign-in` with `rel=nofollow`.
- `Notice-Meta_description_too_short.csv`, `Warning-Noindex_page.csv`, and `Notice-Noindex_follow_page.csv`: `/account/sign-in` is non-indexable with a 90-character description.
- `Warning-3XX_redirect.csv`, `Notice-HTTP_to_HTTPS_redirect.csv`, and `Notice-Redirect_chain.csv`: canonical domain and HTTP-to-HTTPS redirects.
- Change notices for H1/title/meta/word-count appear to be freshness deltas, not defects by themselves.
- `Notice-Pages_to_submit_to_IndexNow.csv`: 78 indexable URLs identified for submission; operational action only unless repo defects are found.

## Planned Fix Areas

1. Link integrity and UX

   - Replace dead `href="#"` links in utility/footer/mobile surfaces with real internal routes, `mailto:` links, or non-link controls where appropriate.
   - Ensure wholesale/contact links reach a stable page or mail target.
   - Keep account sign-in accessible while preserving intentional noindex handling.

2. Image payload control

   - Tighten `Photo` usage where default image sizes over-request large Shopify assets.
   - Add explicit `sizes` for PDP gallery, thumbnails, brand hero/cards, and any repeated product-card contexts that need smaller responsive candidates.
   - Preserve image clarity on desktop PDP hero and mobile product cards.

3. Product discoverability

   - Investigate why the BAL Coffee product has no internal inlinks despite being indexable.
   - Check whether search/category/brand surfaces fetch and render all live products needed for internal discovery.
   - Fix repo-side routing/linking if a product is present in Shopify data but absent from storefront grids.

4. Hardcoded date audit

   - Report user-facing hardcoded dates.
   - Replace the footer copyright year with a runtime/current year value if it is hardcoded.
   - Treat brand founding years and Shopify CDN version query strings as catalog/technical data, not freshness defects.

5. Responsive QA
   - Desktop pass: home, search, major collections, brand pages, product PDP, cart, account entry, policies/reviews/content pages.
   - Mobile pass: nav, category bar, mobile menu, product grids, PDP gallery/options/cart drawer, footer/newsletter.
   - Check visible overlaps, clipped fixed elements, text overflow, dead controls, console errors, and broken navigation.

## Fixes Implemented

### Link Integrity

- Replaced dead utility-bar links:
  - `Gift Cards` -> `Gift Boxes` at `/collections/gift-boxes`.
  - `Become a Partner` -> `/#wholesale`.
  - Removed fake `Refer a Friend · Get $20`; replaced with `/reviews`.
  - `Track Order` -> `/account/orders`.
  - `Help` -> `/shipping`.
  - `USD · EN` is now non-link text.
- Replaced footer `href="#"` links:
  - Social placeholders are non-link badges until real social URLs exist.
  - Wholesale links use encoded `mailto:` URLs.
  - Help links point to existing `/contact`, `/shipping`, and `/policies/refund-policy`.
  - Privacy/Terms point to real policy routes.
  - Accessibility opens a support email.
- Updated the mobile menu utility links to match the real desktop destinations.
- Updated the category bar `Wholesale` link to `/#wholesale` so it works from subpages.
- Fixed the homepage brand header link from a misleading `All N brands -> /brand/zaitoune` to `Shop all brands -> /search`.
- Converted dead brand CTA buttons into real wholesale mail links.
- Converted the gift-box promo `Start Building` button into a real link.
- Fixed duplicate mailto subject construction in the wholesale catalog.
- Removed the repo-owned newsletter `$10 off` CTA/success message and replaced them with non-discount signup copy.

### Interaction Structure

- Reworked product cards so product navigation links and quick-add buttons are separate interactive elements.
  - Previous state: a button was nested inside a product link.
  - New state: image/title regions link to PDP; quick-add is a sibling control.

### Shipping Copy

- Centralized the free-shipping threshold in `lib/babanuj/shipping.ts`.
- Updated repo-owned copy from `$70` to `$75`, matching the Shopify shipping page's live `$75+` policy.
- Updated cart drawer and cart page free-shipping progress math to `$75`.

### Stale Campaign Copy

- Removed the homepage hero's stale `SPRING DROP` / `order before April closes` copy.
- Replaced it with evergreen small-batch maamoul copy.

### Image Payloads

- Added `fallbackWidth` support to the shared `Photo` component.
- Applied smaller optimized fallback `src` widths to product cards, carousels, PDP gallery/thumbnails, promo images, brand hero/cards, and homepage hero images.
- This specifically addresses crawlers that fetch the `src` attribute instead of choosing a smaller `srcset` candidate.

### Judge.me

- Delayed Judge.me widget placeholders until client mount to remove the hydration race caused by third-party DOM injection.
- Changed the widget preloader strategy to `lazyOnload`.
- Added a Judge.me DOM normalizer that:
  - Rewrites injected `/products/<handle>` links to canonical `/product/<handle>`.
  - Rewrites hidden Judge.me `href="#"` attribution links to `https://judge.me`.
  - Rewrites hidden `Write a review` `href="#"` triggers to `#judgeme_product_reviews`.

### Local Dev Stability

- Added `turbopack.root = process.cwd()` in `next.config.ts` so local dev resolves this repo as the root instead of the parent directory when another lockfile exists above it.

## Crawl Report Disposition

- Oversized image file errors: fixed repo-side by reducing Next image fallback `src` widths.
- Orphan / one-inlink product notices: current Chrome crawl discovered product links including the BAL Coffee product from visible storefront/review surfaces. No route-level dead page was found.
- Internal `rel=nofollow` account link notices: intentional for account/noindex entry. Not changed.
- `/account/sign-in` noindex/meta notices: intentional account surface. Not changed.
- HTTP/HTTPS and apex/www redirects: intentional canonical redirect behavior. Not changed.
- IndexNow submission list: operational SEO follow-up, not a repo code defect.
